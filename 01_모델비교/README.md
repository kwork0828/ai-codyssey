# 과제1 - 모델 비교 (상담일지 자동화)

## 1. 비교 개요
- **과업**: 상담 메모 → 정형화된 상담일지(회사 4항목 양식) 자동 변환
- **테스트 대상 모델**: Claude Opus 4.8 / GPT 5.5 / Gemini 3.1 Pro
- **입력 조건**: 동일한 시스템 프롬프트 + 동일한 상담 메모(14기 김OO, 파이널컨설팅)
- **평가 방식**: 4개 평가축 × 각 5점 만점
- **재현성 기록**: 무료(과정 제공 네이토 토큰) 사용, 학습 튜터 모드 동일 적용

## 2. 점수표

| 평가 축 | Claude Opus 4.8 | GPT 5.5 | Gemini 3.1 Pro |
|---|---|---|---|
| ① 상담일지 문체(~함, ~하였음) | 5/5 | 4/5 | 5/5 |
| ② 양식 4항목 준수 | 5/5 | 5/5 | 5/5 |
| ③ 없는 내용 안 지어냄 | 5/5 | 3/5 | 5/5 |
| ④ 한국어 자연스러움 | 5/5 | 5/5 | 5/5 |
| **합계** | **20/20** | **17/20** | **20/20** |

## 3. 항목별 평가 근거

### ① 상담일지 문체 (개조식·보고체)
- **Claude / Gemini (5점)**: 전 항목을 '~함', '~하였음', '~안내함' 등 간결한 보고체로 일관되게 유지함.
- **GPT (4점)**: 대체로 보고체를 지키나 3), 4) 항목에서 문장이 길어지고 서술형에 가까워지는 부분이 있어 감점.

### ② 양식 4항목 준수
- **세 모델 모두 (5점)**: 1) 상담신청내용 2) 현황 3) 상담진행내용 4) 기타 순서와 항목을 정확히 준수함.

### ③ 없는 내용 안 지어냄 (환각 방지) ★핵심 감점 항목
- **Claude / Gemini (5점)**: 메모에 있는 사실만 재구성했고, 원본에 없는 내용을 추가하지 않음.
- **GPT (3점)**: 4) 기타에 "이후 자기소개서 보완 시 ~ 수정하도록 안내함"처럼 **원본 메모에 없는 후속 안내 내용을 임의로 생성**함. 상담일지 자동화에서 가장 위험한 오류(허위 기록)에 해당하여 크게 감점.
  - 또한 상담일지 번호를 지시 없이 '상담일지 2'로 임의 변경함.

### ④ 한국어 자연스러움
- **세 모델 모두 (5점)**: 문장이 어색하지 않고 실무 문서로 바로 사용 가능한 수준.

## 4. 최종 채택 결론

상담일지 작성에는 **Claude Opus 4.8** 모델을 채택함.

- Claude와 Gemini가 20점 동점이나, **상담일지 자동화의 핵심은 '기록의 정확성(없는 내용을 지어내지 않음)'** 이므로 환각 방지 항목에서 만점을 받은 두 모델 중 Claude를 최종 채택함.
- Claude는 원본 메모의 사실만을 충실히 재구성하면서도 보고체 문체와 양식을 완벽히 준수하여, 실무 상담일지 자동화 도구로 가장 적합함.
- 반면 GPT는 문체·양식은 양호했으나 원본에 없는 후속 안내 문장을 임의로 추가(환각)하여, 사실 기록이 중요한 본 과업에는 부적합하다고 판단함.

> Gemini 3.1 Pro도 동등한 성능을 보여 대안 모델로 활용 가능함.
